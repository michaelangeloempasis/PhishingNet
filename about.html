About PhishingNet

Project background, objectives, and development team

Our Mission

PhishingNet was created to combat the growing issue of online phishing attacks by providing users with an intelligent, real-time browser protection tool. The system aims to help individuals recognize and avoid fraudulent websites that attempt to steal sensitive information such as passwords, banking details, and personal data.

Background

Phishing has become one of the most common forms of cybercrime, targeting unsuspecting users through deceptive links and fake login pages. Many victims fall for these scams due to a lack of awareness and poor detection tools. PhishingNet was developed as a capstone project to address this concern by integrating machine learning and semantic text analysis to detect malicious websites instantly through a simple browser extension.

Objectives

  • To design and develop a browser-based phishing detection system using machine learning.
  • To analyze URLs and webpage content to classify whether they are safe or phishing.
  • To notify users in real time whenever a potential phishing attempt is detected.
  • To promote cybersecurity awareness among internet users through visual risk indicators.

How It Works

PhishingNet operates through two main components: a browser extension and a Flask-based backend server. The extension monitors user activity, captures URLs or visible text, and sends them to the backend API. The backend evaluates these inputs using a trained LightGBM model and a MiniLM semantic model to generate a phishing probability score.

  • The extension displays a verdict (SAFE or PHISHING) directly in the browser.
  • Notifications and audio alerts are triggered when phishing risk is detected.
  • All scanning is performed locally to ensure user privacy.

Scope and Limitations

  • The system currently supports Google Chrome and Microsoft Edge browsers.
  • Detection focuses on URLs, page titles, and visible text content.
  • The backend must be running locally for the extension to function.
  • Future work includes cloud deployment and expanded dataset training for improved accuracy.

Beneficiaries

  • General Internet Users: gain real-time protection from phishing links.
  • Educational Institutions: can use the system as a cybersecurity awareness tool.
  • Developers and Researchers: can extend the open API for phishing detection studies.

Technologies Used

  • Frontend: HTML, CSS (Bootstrap-like custom style), JavaScript
  • Backend: Python Flask API
  • Machine Learning Models: LightGBM (URL feature-based), MiniLM (semantic text embeddings)
  • Tools: Visual Studio Code, Chrome Developer Tools, Jupyter Notebook
