System Architecture

A detailed look at how PhishingNet works internally

Overview

The PhishingNet architecture is composed of two major components — the Browser Extension and the Flask-based Backend. Together, they form an intelligent phishing detection system powered by a machine learning model (LightGBM + MiniLM).

Architecture Layers

1. Browser Extension Layer
  • Handles user interaction and data capture (URL, visible text, title).
  • Includes popup UI, service worker, and content script.
  • Communicates with backend via REST API calls to /analyze.
  • Stores scan history and settings in local browser storage.
2. Backend Layer (Flask API)
  • Receives incoming analysis requests from the browser extension.
  • Extracts URL-based features such as domain length, number of dots, HTTPS usage, etc.
  • Processes optional text and title through MiniLM transformer for semantic understanding.
  • Combines features and computes phishing probability using a LightGBM model.
3. Machine Learning Layer
  • LightGBM Model: Trained on large sets of phishing and legitimate URLs.
  • MiniLM Embedding: Converts webpage text into numerical representations for semantic risk analysis.
  • Final score combines URL-based and semantic-based predictions.
4. Storage Layer
  • Uses Chrome Local Storage to store recent scan history, configurations, and user preferences.
  • Ensures privacy — no data leaves the user’s local machine.

Data Flow Process

  1. The user activates the PhishingNet extension and clicks “Scan Now.”
  2. The extension collects the URL, page title, and text (if allowed).
  3. The data is sent to the backend endpoint /analyze.
  4. The backend processes the input using LightGBM + MiniLM models.
  5. The backend returns a phishing risk score and verdict (SAFE / PHISHING).
  6. The extension displays the result and plays a notification sound if phishing is detected.

Key Advantages

  • Lightweight: Minimal impact on browser performance.
  • Accurate: Combines heuristic and ML-based detection for improved accuracy.
  • Private: All analysis is done locally; no external servers involved.
  • Real-time Detection: Scans and displays results instantly.
